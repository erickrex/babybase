# TrailMap

## Brand Direction
TrailMap is practical, information-rich, and action-oriented. It should feel like a modern navigation tool for nomads: clear, dependable, and structured.

## Visual System
- Core palette:
  - Graphite and slate base layers.
  - Topographic greens and amber for emphasis.
  - Clear semantic status colors.
- Surfaces:
  - Utility-first cards with visible grouping.
- Typography:
  - Strong label/value patterns and concise copy.

## Layout System
- Organize by task flow, not decoration.
- Use progressive disclosure for advanced info.
- Keep primary actions sticky or easy to re-find.

## Components
- Data cards with clear titles and metadata rows.
- Filter bars with obvious active/inactive states.
- Timeline and status chips for movement and availability.
- Message and event modules optimized for speed.

## Feature Mapping
- Onboarding: clear input groups and progress states.
- Discovery: card-stack interactions with explicit outcomes.
- Matches/Chat: quick context + fast message actions.
- Campfire: strongest fit for multi-filter event workflows.
- Profiles: structured snapshot (facts, locations, prompts, media).

## Interaction
- Motion should support orientation (where am I, what changed).
- Keep interactions predictable and low-friction.
- Prioritize speed and clarity over visual flourish.

## Implementation Plan
1. Define utility-centric tokens and spacing scale.
2. Standardize data card and filter components.
3. Apply across discovery, matches, campfire, profile.
4. Validate usability in mobile-first flows.

## Quality Checklist
- Information hierarchy is immediately understandable.
- Users can complete key actions with minimal taps.
- Every screen looks like part of one operating system.
