# PacificCoast

## Brand Direction
PacificCoast should feel calm, clean, and premium. It balances cool ocean tones with minimal UI density for a trust-first product feel.

## Visual System
- Core palette:
  - Deep navy/charcoal background.
  - Coastal blue and sea-glass accents.
  - Neutral grays for structure.
- Surfaces:
  - Crisp cards, restrained shadows, strong separation.
- Typography:
  - Minimal, modern hierarchy with high legibility.

## Layout System
- Prioritize simplicity and scanability.
- Keep dense pages modular: one card = one intent.
- Use whitespace as structure; avoid ornamental clutter.

## Components
- Primary actions: clean blue accent.
- Secondary actions: transparent with border.
- Status indicators: semantic but muted.
- Lists and tabs: clear active states with minimal noise.

## Feature Mapping
- Onboarding: clean step framing and controlled copy density.
- Discovery (Friends/Dating): clear card hierarchy and predictable controls.
- Matches/Chat: efficient, polished conversation workflow.
- Campfire: structured event browsing with clear mode segmentation.
- Profiles: editorial layout with strong media grid and quick facts.

## Interaction
- Favor subtle, precise motion over playful motion.
- Keep transitions short and state-driven.
- Emphasize feedback on actions that change system state.

## Implementation Plan
1. Establish cool-toned design tokens.
2. Refactor core components to tokenized styles.
3. Unify all pages to one card/action language.
4. Final accessibility and spacing QA pass.

## Quality Checklist
- UI feels cohesive and professional.
- Complex pages remain easy to scan.
- No tab visually breaks the system.
